Variable Scopes

There are many "scopes" of variables available for use in CFML

Here are some of the most commonly used scopes...

Application

Variables in the application scope are available throughout the application for the life of the application

Server

Available thoughout application code running on the same server

URL

Available throughout application code to reference query string arguments

Session

Available throughout application code during the life of the application. Limited to to a single user's session

Request

Available throughout application code for the duration of the page request. Does not persist beyond a single page load

Variables

Default scope. Scopes variables inside a function or cfc

This

Scopes variables inside a cfc

Arguments

Available inside a function to reference arguments passed to the function.

Local

Used to scope variables inside of a function with the "var" keyword

Form

Available throughout application to reference values POST'ed to a page

Cookie

Available througout application code to reference values of cookies for the application's domain

What if I don't specify a scope?

OpenBD will search for a variable of the same name through the scopes in this order...

  1. Local
  2. Arguments
  3. Variables
  4. CGI
  5. URL
  6. Form
  7. Cookie
